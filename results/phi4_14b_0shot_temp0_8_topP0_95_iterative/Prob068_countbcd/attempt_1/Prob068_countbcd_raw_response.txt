To implement a Verilog module for a 4-digit BCD counter, we'll follow the specifications provided. Each digit will be encoded using 4 bits, and there will be enable signals for each of the upper three digits to indicate when they should be incremented. We'll also include a synchronous active-high reset.

Here's how you can implement this in Verilog:

```verilog
module TopModule (
    input wire clk,
    input wire reset,
    output reg [3:1] ena,  // Enable signals for tens (ena[3]), hundreds (ena[2]), and thousands (ena[1])
    output reg [15:0] q     // Output register containing the four BCD digits
);

    // Internal registers to hold each digit of the BCD counter
    always @(posedge clk or posedge reset) begin
        if (reset) begin
            // Synchronous active-high reset
            q <= 16'b0000_0000_0000_0000;
            ena <= 3'b000;  // Reset all enable signals
        end else begin
            // Handle the ones digit
            if (q[3:0] == 4'd9) begin
                q[3:0] <= 4'b0000;  // Reset to zero after reaching 9
                ena[1] <= 1'b1;      // Enable tens digit increment
            end else begin
                q[3:0] <= q[3:0] + 1;
                ena[1] <= 1'b0;      // Do not enable if not carrying over
            end

            // Handle the tens digit
            if (ena[1]) begin
                if (q[7:4] == 4'd9) begin
                    q[7:4] <= 4'b0000;
                    ena[2] <= 1'b1;    // Enable hundreds digit increment
                end else begin
                    q[7:4] <= q[7:4] + 1;
                    ena[2] <= 1'b0;    // Do not enable if not carrying over
                end
            end

            // Handle the hundreds digit
            if (ena[2]) begin
                if (q[11:8] == 4'd9) begin
                    q[11:8] <= 4'b0000;
                    ena[3] <= 1'b1;    // Enable thousands digit increment
                end else begin
                    q[11:8] <= q[11:8] + 1;
                    ena[3] <= 1'b0;    // Do not enable if not carrying over
                end
            end

            // Handle the thousands digit
            if (ena[3]) begin
                if (q[15:12] == 4'd9) begin
                    q[15:12] <= 4'b0000;
                end else begin
                    q[15:12] <= q[15:12] + 1;
                end
            end

            // Clear enable signals after handling carry-over for all digits
            if (!(|ena)) ena <= 3'b000;
        end
    end

endmodule
```

### Explanation:

- **Reset:** The counter is reset to zero synchronously with the positive edge of `reset`, and all `ena` signals are cleared.

- **Ones Digit (q[3:0]):** It increments every clock cycle. If it reaches 9, it wraps around to 0 and sets `ena[1]` to enable the tens digit increment.

- **Tens Digit (q[7:4]):** Incremented when `ena[1]` is high. If it reaches 9, it resets to 0 and enables `ena[2]`.

- **Hundreds Digit (q[11:8]):** Incremented when `ena[2]` is high. Similar logic applies as with the tens digit.

- **Thousands Digit (q[15:12]):** Incremented when `ena[3]` is high, without reset on overflow since it's the highest significant digit in this module.

The enable signals are used to control which digits should be incremented based on the carry-over from the lower significant digits. The `(|ena)` logic checks if any of the enable signals are set and clears them once all have been processed to avoid unnecessary increments in subsequent cycles.
